The System 1 / System 2 framework is more useful as a design tool than a psychology model. When building interfaces or developer tools, understanding which cognitive system you're targeting changes everything.

Monitoring dashboards that require System 2 (slow, deliberate) analysis get ignored during incidents when engineers are operating in System 1 (fast, automatic). Good alerting works *with* System 1 — pattern recognition, anomaly detection, clear signals.

This directly influenced how I'm thinking about Prism's cognitive layer: surface predictions and anomalies as patterns, not as data requiring analysis.
