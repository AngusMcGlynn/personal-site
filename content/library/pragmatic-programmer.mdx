Read the 20th anniversary edition. Some advice is dated but the core mental models are permanent: tracer bullets, orthogonality, DRY (properly understood as knowledge duplication, not code duplication), broken windows.

The tracer bullet approach — build a thin end-to-end slice first, then flesh it out — is how I start every project now. It's the fastest way to find the hard problems early.
