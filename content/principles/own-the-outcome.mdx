Your job isn't to write code. It's to solve problems. The best solution might be deleting code, using a library, or changing the requirements.

Code you wrote isn't precious. If someone rewrites your module and the system is better for it, that's a win. If a new library makes your hand-rolled implementation unnecessary, delete yours.

Revised to add: this applies to architecture too. Don't defend an architecture because you designed it. Defend it because it's serving the system well, and be ready to change it when it isn't.
